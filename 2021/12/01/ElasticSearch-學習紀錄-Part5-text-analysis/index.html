ElasticSearch 學習紀錄 Part5 - text analysis

Posted on 2021-12-01 Edited on 2021-12-02 In ElasticSearch

前言

這篇文章是出自於線上課程 Complete Guide to Elasticsearch 的所記錄的筆記。

這一篇文章主要介紹 ElasticSearch 搜尋資料的行為。

本文

當資料被紀錄之前，會先透過分析器 analyzer 來處理。其中 analyzer 分成了三部分

  1. Character filters
  2. Tokenizer
  3. Token filters

Analyzer

Character filters

  • 每個 analyzer 都會有零到多個 character filters
  • 用來新增、修改或移除 characters
  • 會因為順序不同而得到不同的結果。

E.g.
I&apos; m in a <em>good mood. -> I’m in a good mood.

Tokenizer

每個 analyzer 都會有一個 tokenizer，作用是幫資料做切分的依據。

E.g.

I do handsome. -> [“I”, “do”, “handsome”]

Token filters

將 tokenizer 所得到的 OUTPUT 當作 INPUT。

  • 用來新增、修改或移除 character
  • 每個 analyzer 可以有零到多個 token filters
  • 會因為順序不同而得到不同的結果

E.g.
lowercase filter
[“I”, “LIKE”, “MINECRAFT”] -> [“i”, “like”, “minecrafe”]

Dates

Elasticsearch 在處理時間的資料會遵循著以下的規範

  • Date 時區會轉換為 UTC
  • 任何格式的 date 在內部皆皆會轉成 long data type 的方式儲存起來
  • 搜尋的條件中若是包含日期，也會自動轉換成 long 的方式查詢
