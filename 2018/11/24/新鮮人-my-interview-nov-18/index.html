新鮮人面試 My Interview at Nov. 18

Posted on 2018-11-24 Edited on 2021-12-03 In Interview

本篇文章節錄 2018.11 面試的公司，分別有

  • 雙子星
  • 科盛科技
  • 趨勢科技

雙子星

應徵職位 : 雲端開發工程師

雙子星透過 104 邀請面試，公司位於新竹京元電子附近，從高鐵站搭小黃前往大約 20 分鐘就能抵達，公司是工研院出來的團隊，是一間蠻新的公司，主要是在做雲端服務相關的產品，透過 openstack 做應用，目前著重在 PaaS 及 SaaS 的開發。

這次的面試過程分成三個階段
第一階段先跟技術協理 Kenneth 做自我介紹，主管也介紹公司目前有三個部門正在找人，分成了   

  • Infra Team: PaaS的開發
  • Software Team: SaaS的開發
  • CAT Team: 跟客戶談需求並解決客戶問題的一個 Team，所以公司內的技術及產品基本上都會碰 (所以說要不排斥面對客戶)

公司會透過你的能力來決定工作崗位，也採取自由的制度轉換工作崗位，而就我面試的狀況，公司也願意讓我自己選擇想要的方向。因為是應屆畢業生，因此公司會建議我從 CAT Team 開始做起，未來也可以根據自己的所好來決定未來的方向。

第二階段就由這三個部門的組長跟我聊 (1 vs. 3 的概念)，主要問了有關過去的經驗，大學的專題、研究所的論文，並沒有問我太多有關雲端的問題，接著就讓我自由提問。

第三階段，主要就跟 HR 經理 Mindy 閒話家常，因為對於薪水的部分在第一階段 Kenneth 就已經有先詢問了，此外也剛好談到北漂的議題，這一階段短短只有10分鐘，也是讓我蠻訝異的，如果有進一步的消息會再通知。

11/29 二面。

這次的面試只是跟 CAT Team Lead - Zero 進一步的對談，在一面所見到的是 CAT Team 底下的一員，主要是再進一步的告訴我 CAT Team 這一個團隊主要的工作範疇。

結果: Offer get

科盛

應徵職位 : Internet 程式設計師

在新竹的台元科技園區，面試總共分成兩個階段

第一階段 HR 會先請你寫 性向測驗 及 英文選擇、選項填空、翻譯題，不難，但是有特定的片語題目，一翻兩瞪眼的題目。

第二階段，應該是主管面試吧，但是主管讓我感到專業度相當不夠…，因為這是我第一次聽到 HTML 是網路程式，發問的回答也讓人感到不受到尊重。

主要了解到這個開發部門分成了五個 Team，而這個職位是第四個 Team。

你說你是管人的，專業度不夠倒還好，但是對待面試者的口氣就會讓人覺得很不恰當。

Fine，要去面試的要慎重考慮嚕。

趨勢科技

應徵職位 : Python Developer

目前在網路上找不到這一個職缺，主要是 104獵頭 主動透過信件向我詢問的

相信讀資訊領域的應該都聽過趨勢科技，而難得有這一個機會，工作內容也跟我找的很相似，當然是二話不說地直接答應囉!

在台北找工作最大的好處是，基本上面試的公司，捷運都能到，相當的便利，而這一次的面試主要分成兩個階段。

第一階段，如同我上面所說的，104獵頭主動詢問我意見，所以一開始獵頭(後面簡稱 Hunter) 告知我趨勢有這一個工作職缺，問我有沒有這一個意願前往面試，在談話的過程中，問到了我許多有關程式開發的想法，這個就根據每個人經驗來回答囉，而 Hunter 也告知我， 要我先到 Codility 進行測試，完成測試之後會再進一步幫我通知該部門，並等待面試通知。

Codility 測驗內容不難，當時我所拿到的題目，主要透過 list 及 一點邏輯就能完成，主要考量的是，Codility 還會評估你的程式碼寫得有沒有效率! 也就是說，測資是有設計過的，所以不僅測驗你程式碼的準確率，還有時間複雜度。

凡事都是 計畫敢不上變化的，在原本規劃面試的行程中， 收到了 趨勢科技 下一個階段的面試邀約，來得太突然，導致原本當天面試完畢要回高雄的我，就直接轉戰台北了。

公司在大安區，台北捷運真的很方便! 

第二階段是跟趨勢科技的 技術經理 Jo面談，主要告訴我這一個職缺在做什麼，不如以往大家所了解的趨勢科技就是做資安寫防毒的，這部門不做資安，不搞大數據，做的是產品的 license ，因為趨勢最近收購了一間美國的公司 tipping point，除了要維持原本美國那裏客戶的服務，同時間也要將此產品做新的開發。

而相對的也會透過 Python 的框架 Django 進行網頁的開發，除此之外還要處理客戶所提出的 bug，進行修復的工作。

為了測試我是不是真的了解 Python，Jo 簡易地問了我 list 中 append 及 extend 的差異性，每次自信的回答都會被對方的質疑而有所猶豫xD，這部分當然還是跟熟悉度有關。

簡易回答一下這個問題
a = [1, 2], b = [3, 4]
a.append(b)  
a = [1, 2, [3, 4]]
a.extend(b)
a = [1, 2, 3, 4]

除此之外還問到了一些

  • Encode vs. Encrypt
  • Encrypt vs. Hash

Jo 也在這一次的面談中，也詢問了很多團隊合作的問題，並強調溝通的重要性，因為這一個部門在 2017.11 剛成立，所以該部門的運作相當的迅速，除此之外也要不排斥用英文溝通，因為還有很多問題是從美國的客戶來的，要根據相對應的問題而做出解決方案。

整個過程聊得蠻愉快的，只不過因為該團隊目前太忙了，所以當我進一步想要了解下一階段面試的時間，Jo 也沒有辦法跟我保證，剩餘的部分若是有下一階段就等我待補充囉!

11/28 二面

二面遇到了更多的團隊成員，包含了一面的 Jo，還有 QA Lead Donna 及 Senior Engineer Henry。

過程中主要問到了 情境處理，也就是看你 Debug 的思維，還有 白板題

Input: lst = [[‘A’, 1], [‘B’, 1], [‘A’, 2], [‘C’, 1], [‘B’, 2]] 
Output: ‘A’:[1, 2], ‘B’:[1,2], ‘C’:[1]  - value要排序

除此之外，也有進一步談到在一面時被問到的問題 xD，因為當時回答得並不完整。還有被問到了對於 細心、學習、溝通 的能力進行自評，這個就根據自己實際的狀況來回答囉!
